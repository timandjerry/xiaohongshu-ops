# 小红书运营战略指南（通用框架）

## 这个 Skill 解决什么问题

账号运营和「写好一篇笔记」是两件事。一篇笔记写得再好，如果**选题方向错、发布节奏乱、定位摇摆、踩了违规红线**，也起不来。

本 Skill 主管**账号级的战略层**：定位、选题、节奏、复盘、涨粉、变现、合规。核心框架通用，适用于任何赛道；具体到「生成某篇可发布的笔记（文案 + 封面 + 内页图）」，交给 `xiaohongshu-bao-kuan-generator` skill 执行——两者分工，不重复。

## 赛道适配（通用框架 + 赛道参考包）

**核心框架（诊断、规划、复盘、涨粉、合规）是通用的，适用任何赛道。** 但每个赛道有自己适配的：内容矩阵、变现路径、爆款案例、专属红线。这些放进**赛道参考包**里，按需加载。

**当前内置参考包**：
- `references/niche-ai-tools.md`（AI 工具 / 编程 / 效率，默认赛道）
- `references/niche-beauty.md`（美妆护肤）
- `references/niche-fashion.md`（穿搭时尚）
- `references/niche-food.md`（美食：探店+食谱）
- `references/niche-home.md`（家居生活）
- `references/niche-maternity.md`（母婴育儿）

工作机制：
- 用户指定赛道 → 读对应的 `references/niche-<赛道>.md`。
- 用户没指定赛道 → 默认按 AI 工具赛道，读 `references/niche-ai-tools.md`。
- 用户指定其他赛道但参考包不存在 → 用通用框架现场推导，并提示「这个赛道还没有专属参考包，建议后续补一个」。
- **扩展新赛道**：复制任一 `niche-*.md` 的结构，替换成新赛道内容，存为 `references/niche-<赛道>.md`，即完成扩展，**无需改 SKILL.md**。

**当前阶段目标：优先涨粉**。主理人现阶段以涨粉为第一目标，变现靠后——选题优先做能拉新曝光、能立人设带来关注的内容。

## 核心工作流

运营是一个闭环，不是一次性输出。按顺序推进，每一环为下一环服务：

```
① 诊断  →  ② 规划  →  ③ 执行（移交 generator）  →  ④ 复盘  →  ⑤ 迭代
现状盘点    选题/排期    单篇内容生成              看数据     调整再循环
```

不是每次都走完全流程。用户问「怎么选题」停在第②步；问「这篇为什么没流量」从第④步切入。先判断用户站在闭环哪一步，再回应。

## ① 诊断：先把账号盘清楚

任何建议都建立在现状之上。进入规划前，先搞清楚这几项（用户没主动给的，问一句或合理推断，别凭空假设）：

| 维度 | 要确认的事 |
|------|-----------|
| 定位 | 一句话能说清「我是谁、给谁看、提供什么价值」吗？说清才算定位成立 |
| 人设 | 真实身份标签，决定信任感 |
| 赛道 | 大赛道下的细分，越聚焦越好起量（赛道画像见对应参考包） |
| 现状 | 粉丝量、近 10 篇平均数据、有没有出过爆款 |
| 资源 | 每周能更几篇、能不能出镜、有没有独家信息源 |
| 目标 | 涨粉？接广告？卖课？先有目标，选题才知道往哪使劲（当前默认：优先涨粉） |

**定位自检**：让用户用一句话介绍自己账号。如果一句话说不清、或者说完不知道给谁看，那定位本身就是问题，先解决定位再谈选题。

## ② 规划：选题与内容矩阵

这是本 Skill 最核心的产出。好的运营 = 持续产出**有方向、有节奏、不内耗**的选题。

### 选题优先级打分

拿到一堆选题想法后，按三个维度排序（当前阶段「涨粉价值」权重最高）：

- **涨粉价值（当前优先）**：能拉新曝光、能立人设让人愿意关注吗？实战案例、清单合集、热点速递、系列化内容涨粉最强；纯教程涨收藏但不一定涨粉。
- **搜索量**：小红书搜索框输入关键词，下拉联想词多不多。有人搜才有长尾流量。
- **竞争度**：搜出来前排是不是大号霸榜。找「大号没覆盖但有人在搜」的缝隙。

**高分题公式**：涨粉价值高 × 搜索量大 × 竞争适中 = 优先做。三者都不沾的题，再有趣也先放着。

### 内容矩阵

健康账号不是只发一种内容。每个赛道有适配的内容矩阵——几类内容按比例组合，兼顾「涨收藏的干货底座」和「拉流量的互动入口」。

- 各赛道（美妆/穿搭/美食/家居/母婴/AI 工具）的具体矩阵见对应 `references/niche-<赛道>.md`。
- 用户指定赛道就读对应参考包；没有参考包就按通用思路（教程 / 测评 / 技巧 / 实战 / 资讯 / 合集 的变体）现场推导。

**为什么必须做矩阵**：纯干货账号流量天花板低、枯燥；纯蹭热点账号没沉淀、留不住粉。矩阵让账号既有干货底座又有流量入口。

### 选题库与排期

规划时直接给一张**未来 1-2 周的选题表**：

| 日期 | 选题 | 内容类型 | 目标（涨粉/收藏/变现） | 关键词 |
|------|------|---------|----------------------|--------|
| 周一 | （示例） | 教程 | 收藏 | — |
| 周三 | — | 测评 | 互动 | — |
| 周五 | — | 技巧 | 收藏+传播 | — |

**节奏建议**：新人 2-3 篇/周，固定时间发，让算法和粉丝养成预期。宁可少而稳，不要爆更后又断更。

### 选题避坑

通用红线见「⑥ 合规」。各赛道的专属选题雷区见对应参考包（如 AI 工具赛道的灰色技术教程红线、美妆赛道的功效宣称红线等）。

## ③ 执行：移交 generator skill

选题和方向定下来后，**单篇内容的生成（文案 + 封面 + 内页图）交给 `xiaohongshu-bao-kuan-generator` skill**。本 Skill 不重复造轮子。

### 移交时必须带的「账号风格规范」

generator skill 的默认设定和主理人实战偏好有出入。移交时**以本规范为准**，明确告诉它覆盖默认值（这套规范跨赛道通用）：

| 规范项 | 主理人要求 | 原因 |
|--------|-----------|------|
| 标题字数 | **≤ 20 字** | 超过会被平台折叠截断，首页只显示前十几个字 |
| emoji | **克制使用**，口语化优先 | 少 emoji、去 AI 味，是已验证的调性 |
| 正文字数 | **500-600 字** | 精简、完读率高，读者没耐心看长文 |
| 配图 | 优先 `huashu-design`，降级用 HTML 卡片 | baoyu 系列不稳定，HTML 卡片是可靠兜底 |
| 合规 | 规避引流话术 + 赛道专属红线 | 详见「⑥ 合规」与赛道参考包 |

移交示范话术：「用 generator skill 生成这篇笔记，标题控制在 20 字内、少用 emoji、正文 500-600 字、规避敏感词。」

## ④ 复盘：从数据里找答案

笔记发出去不是结束。运营能力 = 复盘能力。每篇发布 48 小时后看一次数据。

### 看哪些指标（小红书创作者中心）

| 指标 | 看什么 | 健康信号 |
|------|--------|---------|
| 曝光量 | 流量停在第几个池 | 200-500 初始 / 3000-5000 小 / 1w-2w 中 / 10w+ 大 |
| 点击率 | 封面+标题行不行 | <5% 偏低，>10% 优秀 |
| 完读率 | 开头钩子和正文行不行 | 低说明前 3 行没留住人 |
| 收藏率 | 干货密度行不行 | 收藏是算法权重最高的互动之一 |
| 评论率 | 互动设计行不行 | 评论权重高（4 分），低说明没勾起表达欲 |
| 涨粉 | 内容是否带来沉淀 | 爆款不涨粉 = 内容好但人设没立住 |

### 问题诊断对照表

| 症状 | 病根在哪 | 怎么改 |
|------|---------|--------|
| 曝光低 | 没进流量池 / 触发限流 | 查敏感词、查是否被判营销号、查首发互动 |
| 曝光高但点击低 | 封面/标题不行 | 下篇重点改标题（≤20字、有数字/痛点）和封面 |
| 点击高但完读低 | 开头没钩子 | 重写前 3 行，先抛冲突或结果 |
| 完读高但收藏低 | 干货不够「必须存」 | 加清单/步骤/模板/对比表 |
| 收藏高但评论低 | 缺互动钩子 | 结尾加一个一句话能答的问题 |
| 数据好但不涨粉 | 人设模糊 | 在正文和主页强化「我是谁」 |

### 复盘输出格式

```
【这篇数据】曝光 X / 点击率 X% / 收藏 X / 评论 X / 涨粉 X
【流量层级】停在哪个池，是否正常
【做得对的】1-2 条，强化它
【问题在哪】对照上表，定位到具体环节
【下篇怎么改】1-3 条可执行动作
```

## ⑤ 涨粉与互动

### 评论区运营

评论区是算法第二战场（评论权重 4 分）。每篇发布后：

- **前 30 分钟盯评论**，有人评论立刻回——前期互动密度影响算法是否推流。
- **预埋 1-2 条评论**引导方向（补充信息、抛个问题），但**绝不用来引流**（私信/加群/留联系方式 = 违规）。
- 高赞评论截图当下一篇的选题素材——用户的真实痛点是最好的选题源。

### 涨粉的底层逻辑

粉丝关注的是「**持续提供价值的那个人**」，不是某一篇爆款。所以：

- **主页统一**：头像、简介、置顶笔记都在回答「关注我能得到什么」。
- **系列化内容**：把零散选题串成系列，给关注理由。
- **人设稳定**：别今天这个赛道明天那个，定位漂移最伤粉。

## ⑥ 合规（重点：安全合规优先）

主理人踩过坑——一篇笔记因为结尾「评论区留言或私信获取」被判违规。所以本 Skill 的合规原则只有一条：

> **不教钻空子，不靠谐音/图片规避。直接避开红线，用合规方式达成目的，或干脆不做。**

### 绝对红线（全平台通用）

1. **引流话术**：「私信我」「评论区留言获取」「加微信/群」「留联系方式」「看主页简介」「扫码」。→ 资料交付走小红书店铺/专栏合规发，或被动等粉丝私信，不在笔记里设钩子。
2. **绝对化用语**：第一、最、顶级、全网首发、销冠（违反广告法）。
3. **站外链接、二维码、外部平台名**。
4. **医疗/金融类**无资质不碰。

### 引流的合规替代方案

想要资料交付或建立联系，但笔记里不能引流：

| 想达成 | 合规做法 |
|--------|---------|
| 发教程资料/模板 | 用小红书「店铺」或「专栏」合规交付，或直接把干货写进正文 |
| 让粉丝联系你 | **被动等粉丝私信你**（合规），而不是你写「私信我」 |
| 引导关注 | 写「关注我，持续分享 XX」即可，**不带获取动作** |
| 建立信任 | 靠内容质量和人设，不靠引流钩子 |

> 钩子（评论领、私信领）是违规重灾区。把价值直接放进正文，让笔记本身值得收藏，比设钩子更安全也更有效。

### 赛道专属红线

不同赛道有额外红线（如 AI 工具赛道的翻墙/镜像/破解教程、美妆赛道的功效宣称、母婴赛道的婴幼儿食品宣称等）。见 `references/niche-<赛道>.md` 的「专属红线」部分。通用红线完整词库见 `references/banned-words.md`。

## ⑦ 变现

有流量后要想清楚怎么变现。变现路径因赛道而异：

- 各赛道（美妆/穿搭/美食/家居/母婴/AI 工具）的具体变现路径见对应 `references/niche-<赛道>.md`。
- **通用原则**：先内容后人设再变现；粉丝 <1000 时别急，打磨内容和数据模型为主；变现前置条件是内容和人设立住。

**当前阶段优先涨粉，变现靠后。**

## 使用约束

- **不重复 generator 的活**：本 Skill 不写单篇文案正文、不做封面图，那是 generator 的职责。
- **不教违规技巧**：任何谐音规避、引流话术、钻审核空子，一律不给。安全合规优先是硬原则。
- **基于现状给建议**：不假设用户的粉丝量、数据、资源。信息缺失先问。
- **风格规范以本 Skill 为准**：标题 ≤20 字、少 emoji、正文 500-600 字、规避敏感词——跨赛道通用，移交 generator 时带上覆盖其默认值。
- **赛道参考包按需加载**：用户说哪个赛道就读对应 niche 参考包；没有就用通用框架推导并提示补充。
