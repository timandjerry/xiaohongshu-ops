# 小红书运营战略 Skill(xiaohongshu-ops)

> 小红书账号运营的「战略大脑层」:定位 / 选题 / 复盘 / 涨粉 / 变现 / 合规。
> 通用框架 + 赛道参考包,适用于任何赛道,默认带 AI 工具赛道。

账号运营和「写好一篇笔记」是两件事。一篇笔记写得再好,如果**选题方向错、发布节奏乱、定位摇摆、踩了违规红线**,也起不来。这个 Skill 主管**账号级的战略层**;至于「生成某篇可发布的笔记(文案 + 封面 + 内页图)」,交给配套的 [`xiaohongshu-bao-kuan-generator`](#与-generator-skill-的分工) 执行——两者分工,不重复。

---

## 核心能力(战略层六件事)

| 职责 | 解决什么 |
|------|---------|
| **定位 / 人设** | 一句话说清「我是谁、给谁看、提供什么价值」,定位成立才谈选题 |
| **选题规划** | 选题打分排序、内容矩阵、未来 1-2 周排期表,持续产出有方向有节奏的选题 |
| **数据复盘** | 看曝光/点击/完读/收藏/评论/涨粉,用问题诊断对照表定位病根 |
| **涨粉策略** | 评论区运营、主页统一、系列化内容,把流量沉淀成关注 |
| **商业变现** | 按赛道给变现路径(当前阶段优先涨粉,变现靠后) |
| **违规规避** | 绝对红线 + 合规替代方案,不教钻空子 |

## 设计:通用框架 + 赛道参考包

**核心框架(诊断、规划、复盘、涨粉、合规)是通用的,适用任何赛道。** 每个赛道独有的内容矩阵、变现路径、爆款案例、专属红线,放进**赛道参考包**按需加载。

工作机制:
- 用户指定赛道 → 读对应的 `references/niche-<赛道>.md`
- 没指定赛道 → 默认按 **AI 工具赛道**
- 指定了赛道但参考包不存在 → 用通用框架现场推导,并提示「建议后续补一个」
- **扩展新赛道**:复制任一 `niche-*.md` 结构,替换内容存为 `references/niche-<赛道>.md` 即可,**无需改 SKILL.md**

---

## 运营闭环

运营是一个闭环,不是一次性输出。按顺序推进,每一环为下一环服务:

```
① 诊断  →  ② 规划  →  ③ 执行(移交 generator)  →  ④ 复盘  →  ⑤ 迭代
现状盘点    选题/排期    单篇内容生成              看数据     调整再循环
```

不是每次都走完全流程。问「怎么选题」停在第②步;问「这篇为什么没流量」从第④步切入。Skill 会先判断你站在闭环哪一步,再回应。

## 怎么用

这是一个 [Claude Code](https://docs.claude.com/en/docs/claude-code) Skill。把它放进 `.claude/skills/xiaohongshu-ops/` 目录后,Claude 会在你聊到相关话题时自动触发,也可以直接点名调用。

**会触发它的说法**:
- 「帮我做个选题规划」
- 「我的笔记被限流了」
- 「怎么涨粉」
- 「这个赛道能变现吗」
- 「我的账号定位有没有问题」
- 「帮我复盘一下最近的数据」

**给个具体的赛道**会得到更贴合的结果,例如「我在做 AI 工具赛道 / 美妆赛道,帮我规划下个月的选题」。

## 与 generator skill 的分工

| | xiaohongshu-ops(本 Skill) | xiaohongshu-bao-kuan-generator |
|--|---|---|
| 层级 | 战略层 | 执行层 |
| 产出 | 定位、选题表、排期、复盘诊断、涨粉策略 | 单篇可发布的文案 + 封面 + 内页图 |
| 关系 | 规划方向后**移交**给 generator 执行 | 接收方向,生成具体内容 |

移交时会把**账号风格规范**一并带上,覆盖 generator 的默认值:

| 规范项 | 要求 | 原因 |
|--------|------|------|
| 标题字数 | ≤ 20 字 | 超过会被平台折叠截断 |
| emoji | 克制,口语化优先 | 去 AI 味,已验证的调性 |
| 正文字数 | 500-600 字 | 精简、完读率高 |
| 配图 | 优先 `huashu-design`,降级 HTML 卡片 | 可靠兜底 |
| 合规 | 规避引流话术 + 赛道专属红线 | 详见 SKILL.md「⑥ 合规」 |

## 合规原则

主理人踩过坑——一篇笔记因为结尾「评论区留言或私信获取」被判违规。所以合规原则只有一条:

> **不教钻空子,不靠谐音 / 图片规避。直接避开红线,用合规方式达成目的,或干脆不做。**

绝对红线包括:引流话术(私信我 / 加群 / 留联系方式)、绝对化用语(第一、最、顶级)、站外链接与二维码、无资质的医疗 / 金融内容。完整词库见 [`references/banned-words.md`](references/banned-words.md)。

## 当前阶段目标

**优先涨粉,变现靠后。** 选题优先做能拉新曝光、能立人设带来关注的内容;粉丝 <1000 时不急着变现,打磨内容和数据模型为主。
